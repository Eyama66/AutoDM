# AutoDM 项目概览

AutoDM 当前是一个可运行的 Web POC。它已经把 AI 叙事、离散动作协议和中文模组数据打通，但文档以下内容只描述当前已落地的部分。

## 架构现状

| 层次 | 技术选型 | 核心职责 |
| :--- | :--- | :--- |
| 表现层 | React 19, Tailwind, Framer Motion | 沉浸式 UI、角色面板、检定按钮、战斗操作 |
| 核心层 | TypeScript | 地理移动校验、属性更新、变量更新、基础战斗与检定计算 |
| AI 适配层 | OpenAI SDK 兼容接口 | 根据当前区域上下文生成叙事与 `@ACTION` 标签 |
| 本地存档层 | `localStorage` | 保存角色卡、聊天记录、区域位置与剧情变量，支持刷新恢复 |

## 当前状态

1. 前端可运行：`web-client` 可直接 `npm run dev` 和 `npm run build`。
2. 规则核心可运行：`ActionProcessor`、`CampaignManager`、`CombatEngine`、`CoreRules` 已能联动。
3. 中文检定映射已修正：中文技能名与能力检定不再默认回退到力量。
4. 本地会话恢复已接通：刷新后可恢复角色卡、区域位置、消息历史与变量。
5. 核心测试脚本已修复：`core` 的类型检查和两条脚本化验证链路可直接运行。
6. 服务端化 AI 调用仍未完成：生产级的后端代理、会话隔离和流式返回尚未接入。

## 下一步

- 把浏览器侧 LLM 调用迁到 `server/`，避免 API key 暴露。
- 补全战斗回合流转、敌方行为和胜负结算。
- 把 `docs/schema` 里的 JSON 规范整理成真正可校验的 schema + loader。
- 再考虑流式返回、多用户会话和更完整的剧情状态机。
